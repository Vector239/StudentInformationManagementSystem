# 学生信息管理系统

- 前端：`cd frontend && npm run serve`
- 后端：`cd backend && uvicorn app:app --reload --host 127.0.0.1 --port 8000`

---

## 一、核心原则

适合写进简历 + 能部署到 GitHub 的学生信息管理系统，要满足「功能完整但不冗余」「有技术亮点」「能体现你的技术栈」。按「核心必备功能 + 进阶加分功能」梳理，每个功能都标注「简历怎么写」「GitHub 展示亮点」。

## 二、完整功能清单（前后端分离版）

### （一）核心必备功能（必须实现，体现基础能力）

#### 1. 用户认证模块（简历核心亮点：权限控制、安全校验）

| 功能点 | 具体实现 | 简历怎么写 | GitHub 展示 |
|--------|----------|------------|-------------|
| 登录 / 注册 | 支持手机号 / 账号密码登录，验证码校验，密码加密存储（bcrypt） | 实现基于 JWT 的用户认证机制，密码采用 bcrypt 加盐哈希存储，防止明文泄露 | 展示登录页截图 + 核心加密代码片段 |
| 权限分级 | 超级管理员（增删改查所有数据）、普通管理员（仅查看 / 编辑）、学生（仅查看自己信息） | 基于 RBAC 模型实现三级权限控制，不同角色可见 / 可操作的功能模块隔离 | 展示权限控制流程图 + 角色菜单对比截图 |
| 密码找回 | 邮箱 / 手机号验证码找回，重置密码 | 实现安全的密码重置流程，验证码有效期控制（5 分钟） | 展示找回密码的交互流程 |
